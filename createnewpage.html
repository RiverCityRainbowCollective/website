Adding a Page to the Website

Website File Structure

If you look at the sidebar on the left, you'll see a website folder. This folder contains all the markdown (.md) files that will end up on the website. Everything outside of that folder will not be included.

The structure of the files inside the website folder are hierarchical by category. This hierarchy will become the navigation inside the website.

 

Only Markdown files are supported on the website!

All files inside the website folder should be created via the Markdown text editor. The website software we use accepts markdown files and renders them into HTML files before the website is fully deployed.

 

Step by Step Instructions

  1. Click on the Create new Page button in the top right toolbar

2. In the Select New Page Location modal, navigate to the “virtual folder” where you'd like your page to exist. If the folder you want does not exist, you can edit the file path of the page and add a new folder to the directory like this:

/website/resources/healthcare/new-folder-name/new-page-name

Press Select when you are finished

3. Choose Markdown Editor. The next modal will ask which editor you want to use. The way the website is currently structured, it will only accept markdown formatted files. If you select another editor, it won't break anything, but your page will not show up in the website. If you happen to make a page in a different editor, there is an option somewhere to convert it to markdown, but with a potential loss of formatting.

4. Edit the Page Properties then select OK when ready

5. Add your Markdown formatted content then press CREATE. Click the ? icon on the sidebar to see a markdown formatting reference guide.
